# 公众号排版规范 v4

> 每日AI资讯的公众号文章排版标准。
> 风格：极简杂志风 — 干净、疏朗、视觉统一。

## 排版参数

| 参数 | 值 |
|------|-----|
| 正文字号 | 15-16px |
| 行高 | 1.75倍 |
| 段落间距 | 段与段之间空一行 |
| 首行缩进 | 不缩进 |
| 两端留白 | 左右各 16px |
| 配色主题 | 深蓝 #2B6CB0 / 暗红 #C53030 |
| 引用/金句 | 加粗 + 深蓝色 |
| 分隔线 | `—— ✦ ——` （居中，灰色） |

## 文章标题（外部显示）

标题独立于正文，是公众号外部列表展示的标题，**不要**写在正文第一行。

**格式：**
```
YYYY年MM月DD日 - [独立标题，15-25字]
```

**示例：**
```
2026年06月07日 - 当AI开始设计AI，开发者还剩下什么？
2026年06月07日 - 给AI一家实体店，它撒了谎
```

## 正文结构

### 模板（含配图）

```markdown
![封面描述](D:/opc-workdown/outputs/cover-YYYY-MM-DD.png)

🟣 今日速览
• 第一篇一句话
• 第二篇一句话

—— ✦ ——

▍ 标题一
📌 一句话钩子

![配图描述](D:/opc-workdown/outputs/article-关键词-YYYY-MM-DD.png)

正文段落。每段不超过 3 行，段与段之间空一行。关键数据或金句加粗。

💡 收尾

—— ✦ ——

▍ 标题二
...
```

### 图片规则

| 位置 | 数量 | 尺寸 | 来源 |
|------|:----:|:----:|------|
| 封面（文章顶部） | 1张 | cover_wechat (1024×512) | siliconflow_gen.py --ratio cover_wechat |
| 每篇文章（📌之后） | 1张/篇 | landscape_16_9 (1024×576) | siliconflow_gen.py --ratio landscape_16_9 |

- 图片用 `![描述](路径)` 直接嵌入，**不要加任何前缀/后缀文字**（如"🎨 配图"、"📡 硅基流动生成"）
- 封面前后不加文字说明
- 文章配图放在 📌 之后、正文之前

### 每篇文章结构

| 元素 | 格式 | 说明 |
|------|------|------|
| 小标题 | `▍ 标题` | ▍ + 空格 + 标题，加粗 |
| 钩子 | `📌 一句话钩子` | 单独一行 |
| 配图 | `![描述](路径)` | 钩子之后、正文之前，landscape_16_9 |
| 正文 | 普通段落 | 300-500字，每段≤3行，段间空行 |
| 金句/数据 | **加粗** | 关键数字/结论加粗 |
| 收尾 | `💡 一句话收尾` | 直接写观点，不加"复盘式/延伸式/对比式"分类词 |

### 分隔规则

- 文章之间用 `—— ✦ ——` 分隔，居中
- 正文与底部引导之间用 `—— ✦ ——` + 空行 + `---`
- 不在正文中使用纯符号分隔线（如 ━━━━━━）

## 注意事项

- 每天 5-7 篇，同主题合并
- 纯中文，不出现英文标题
- 不要评分（⭐）、不要来源链接（🔗）、不要标签（#）
- 💡 收尾直接用观点，不要加"复盘式/延伸式/对比式"分类词
- 正文用通俗语言讲技术红利，不需关联一人公司
- 每篇正文字数 300-500 字
- 所有图标统一使用：🟣 📌 💡 ✦
- 图片路径用绝对路径：`D:/opc-workdown/outputs/...`
