# 写作风格指南 — 袁兜兜

> 更新时间: 2026-06-14 | 新增: 定位校准, 冷启动选题规则, 口播稿规范

## 核心原则: 去AI味, 要真人味

用户批评过"AI味太重了"——具体要求如下:

### 要避免的

- ❌ "范式转移" "降维打击" "底层逻辑" 等大词
- ❌ 工整的模板式三段落（第一点...第二点...第三点...）
- ❌ 每段结尾都塞金句
- ❌ 通用模板式的开头（"随着AI技术的快速发展..."）
- ❌ 图片前后加"🎨 配图" "📡 硅基流动生成"等水印文字

### 要采用的

- ✅ 用真实个人经历开头（工作年限、失败的代价）
- ✅ 承认迷茫和痛苦，不包装
- ✅ 口语化语气（"坦白说" "你懂的" "但我觉得不是"）
- ✅ 先抛困惑再给结论
- ✅ 厘清易混淆的概念对比
- ✅ 结尾留互动钩子
- ✅ 图片直接用 `![描述](path)` 嵌入，不加前缀后缀

## 人设定位于文章中的体现

❌ 不写"我是一个有8年经验的Java后端..."
✅ 写"你有没有过这种感觉: 每天打开电脑，处理一堆重复的事情..."

定位公式:
```
不是「程序员教你怎么用AI」
而是「一个每天都在用AI干活的人, 教你怎么也让AI帮你干活」
```

## 选题时的冷启动适配

发布前先问自己:

1. [ ] 一个不认识我的人看到标题会想点进来吗?
2. [ ] 标题有具体问题/数字/冲突吗?
3. [ ] 开头前两句能让人想读第三句吗?
4. [ ] 这篇对不写代码的人也有用吗?

4个都是"是" → 放心发。否则先优化或换选题。

## 公众号排版规范

- **独立标题**: `YYYY年MM月DD日 - [15-25字标题]`，不嵌入正文
- **正文**: 杂志风格 — `🟣 今日速览` → `—— ✦ ——` → `▍ 标题` → `📌 钩子` → 正文 → `💡 收尾`
- **分隔线**: `—— ✦ ——`（不要用 `━━━━━━`）
- **底部引导**: `📮 觉得有启发? 欢迎转发给需要的朋友。`
- **不要**: 评分(⭐)、来源链接(🔗)、标签(#)、"xxx式收尾"分类词

## 口播稿规范

将公众号文章转换为口播稿（voice-over script）时:

1. **短句分行** — 一句话或一个意群一行，方便录音时看断句
2. **去掉视觉标记** — 去掉 ▍、—— ✦ ——、引用框 > 等排版符号
3. **更口语化** — 加入"你懂的""坦白说""我就好奇了"等自然过渡
4. **停顿用标点控制** — 不要用【停顿】【长停顿】等自定义标记，TTS（如VoxCPM）会当成文字读出来。改用句号断句制造自然停顿，空行制造长停顿
5. **声音描述用英文** — VoxCPM 的 voice design 模式用英文描述效果更好，如 `(male, warm and confident, like a podcaster talking to a friend)`

## 用户审阅流程

- 用户会在 outline.md 上直接加批注（如"原文AI味太重了" "删除" "改为..."）
- **不要**把批注合并到最终版本中
- 解析批注意图 → 写干净版本 — 没有批注标记、没有删除线、没有括号说明
- 用户期望拿到可直接发布的文档，不是修订模式

## 飞书推送 vs 公众号文章

- 飞书群推送: 用 `scripts/feishu_push.py`（富文本 post 格式，自动编码）
- 公众号发布: 用 `hermes send --to weixin --subject "标题" --file /path/article.md`
- 注意: 微信 iLink 目前限流不可用，公众号内容先保存到 wechat-push/ 目录
