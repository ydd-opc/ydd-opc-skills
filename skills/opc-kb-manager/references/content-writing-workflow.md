# 写作流程方案

## 两条内容线

### 线一：创业故事（小红书 + 公众号，周更）

```
周一: strategist 选题
  → 搜索同类爆款标题结构
  → 输出 3 个备选标题 + 1 个推荐
  → 给出发布时间建议

周二: researcher 调研
  → 搜索案例、数据、方法论
  → 查询知识库已有素材
  → 输出: 观点大纲 + 素材包

周三: writer 初稿
  → 基于大纲写初稿 (1500-2500字)
  → 保持用户个人风格: 口语化 + 有故事 + 有干货
  → 输出: 初稿 .md → 用户审阅

周四: designer 排版封面
  → 公众号版: 卡片式排版 + 数据可视化 + 头图 (2.35:1)
  → 小红书版: 短段落适配手机 + 3:4竖版封面 + 精简到1000字

周五: 复盘
  → 你发布文章后说"复盘本周"
  → researcher 搜索平台提取公开互动数据
  → strategist 输出: 复盘报告 + 下周选题建议 + 内容优化方向
  → 保存到 strategy/YYYY-MM-DD-复盘.md
```

### 线二：每日AI推文（仅公众号，日更·自动化）

```
每天 9:00 自动触发:
  kb-manager (Horizon)
    → wiki/horizon/日期/ 入库
    → 生成公众号文章（杂志风排版）
    → designer 配封面 + 每篇文章配图
    → 微信推送
```

公众号排版标准见 `references/wechat-push-format.md`（v4 公众号排版规范）。

## 创业故事写作规范（实操经验）

### 处理用户对大纲的修改

用户可能会**直接在 outline.md 文件上编辑**（而不是通过对话告诉你修改意见）。特征：
- 文件修改时间晚于上次写入时间
- 在原文旁边加批注如 `原文(AI味太重了)`、`将XXX改为YYY`、`删除`、`这里不需要方案了`
- 用自己的版本替换原文段落

**应对流程：**
1. 读取 outline.md 前检查文件修改时间
2. 逐段解析用户的修改批注和替换内容
3. 用用户提供的版本替换原文，**删除所有批注标记**
4. 用户的修改优先级最高——即使打乱了原有结构，也以用户版本为准
5. 保存干净版本到 outline.md，再写初稿

### 写作风格指南（去"AI味"）

**用户批评过"AI味太重了"**。以下是明确要避免和要做到的事：

#### ❌ 要避免的（AI味特征）

1. **通用模板式开头**："2026年我失业了。没有刷APP，没有改简历，我做了一个反直觉的决定"——这太像范文模板
2. **过于工整的结构**：每个段落结尾都来一句"金句"或"引发思考"
3. **堆砌大词**："范式转移"、"降维打击"、"底层逻辑"、"认知升级"——用户不说这些话
4. **数据撑场面但没有个人故事**：只用行业数据不用自己经历
5. **假装有温度但其实是套路**："这篇文章不讲鸡汤，只讲三件事"——本身就是一种鸡汤公式

#### ✅ 要做的（真人味特征）

从用户自己改写的版本中提取的风格：

1. **用真实的个人经历开头，越具体越好**：
   - ❌ "失业后没投简历"
   - ✅ "工作8年失业后选择创业，创业2年，创业失败失去了我工作8年所有的积蓄"
   - 具体数字（8年、2年、所有积蓄）远比模糊描述有力

2. **承认迷茫和痛苦，不包装**：
   - ✅ "每天在迷茫和失落中度过"
   - ✅ "不知道自己该往哪走，也不知道还有什么路可以走"
   - 承认脆弱比展示坚强更真实

3. **口语化语气，像在跟朋友聊天**：
   - 用"坦白说"、"你懂的"、"但我觉得不是"这类口语连接词
   - 用短句，段落不要太长
   - 偶尔用反问句跟读者互动

4. **先抛真实困惑再给结论**：
   - ❌ "一人公司是个体系性的机会。以下三点..."
   - ✅ "搜一人公司出来10篇文章，8篇在讲超级个体——好像一人公司就是当网红。但我觉得不是。超级个体是卖自己，一人公司是卖系统。"

5. **给概念做清晰的对比区分**：
   - 用户特别在意概念混淆（一人公司 vs 个体工商户 vs 超级个体）
   - 用"前者...后者..."的对比句式厘清边界

6. **结尾留互动钩子**：
   - ✅ "大家觉得做一人公司第一件事应该干什么呢？"
   - 不要自己把答案说完，留空间给读者评论

### 初稿结构模板（创业故事）

```
引言 — 个人真实困境（为什么走投无路）
  ↓
概念辨析 — 定义关键术语、厘清混淆点（一人公司 vs 个体户 vs 超级个体）
  ↓
主体段一 — 揭露行业泡沫（自媒体上的营销套路）
  ↓
主体段二 — 介绍可信框架（Dan Koe 方法论 + 国内适配）
  ↓
主体段三 — 我的解决方案（个人知识库 + AI Agent）
  ↓
结尾 — 收束全文 + 个人金句 + 互动钩子
```

### 字数控制
- 公众号版：2000-2500 字
- 小红书精简版：800-1000 字，从主体段一和二中提炼

## 目录结构

```
D:\opc-workdown\
├── research/YYYY-MM-DD/     ← researcher 输出（outline.md + materials.md + sources.txt）
├── drafts/YYYY-MM-DD/       ← writer 初稿
├── outputs/                  ← designer 封面/配图
├── strategy/                 ← strategist 选题日历/复盘
└── wiki/horizon/YYYY-MM-DD/  ← kb-manager 每日情报
```
