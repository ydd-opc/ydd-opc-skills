# Strategy Pipeline — 选题策略与自动化推送

> Created: 2026-06-14 | 面向「袁兜兜opc」公众号

## 核心定位

**AI技能教程，面向所有人。**

```
不是「程序员教你用AI」
而是「一个每天都在用AI干活的人，教你怎么也让AI帮你干活」
```

- 不写纯技术教程（Python调API、部署模型）
- 不突出「8年后端经验」在人设中
- 程序员能用 + 普通人也能用 = 及格线

## 选题三问（每篇选题过）

1. **痛点够痛？** — 这个事是不是很多人每天在做？
2. **门槛够低？** — 不写代码的人能做吗？步骤≤5步？
3. **效果够明显？** — 做完能看到立竿见影的效果吗？

## 目录结构

```

├── 选题管线.md         ← 选题队列 (P0→P1→P2→已发布)
├── 选题策略.md         ← 方法论 (定位/公式/矩阵)
├── 内容矩阵.md         ← 四象限分析
└── topic_picker.py     ← 自动选题推送脚本
```

## 自动推送流程

`topic_picker.py` 的逻辑：

1. 读取 选题管线.md → 解析 P0/P1/P2/已发布 表格
2. 按优先级 + 冷启动分 选出最佳选题
3. 检查最近发布类型（连续3篇资讯→警告）
4. 构建富文本消息 → POST 到飞书群 Webhook

### 消息格式

```
📋 本周选题推荐 | YYYY年MM月DD日

━━━━━━━━━━━━━━━━━━━━

🎯 推荐选题：[标题]
📂 类型：[技能/资讯/故事/产品]
⭐ 冷启动分：[N/10]
📎 备注：[原因]

📝 理由：
  • P0 优先级最高，直接开写
  • [冷启动/类型/时效性依据]

━━━━━━━━━━━━━━━━━━━━

💡 收到后直接回复「开写」我就开始写
```

## Cold-Start 适配分级

| 分数 | 含义 | 行动 |
|:----:|------|------|
| 8-10 | 🟢 陌生人看了也想点 | 放心发 |
| 5-7 | 🟡 有吸引力但可优化 | 改标题/封面后发 |
| 0-4 | 🔴 不要发 | 换选题 |

## 用户content偏好（2026-06-14确认）

- ❌ 不要在选题或人设中提「8年后端」「Java转AI」
- ❌ 不要写纯技术教程（面向程序员不面向普通人）
- ❌ 不要教"搭知识库"作为核心定位
- ✅ 核心定位：AI技能教程，任何普通人都能用的实用技能
- ✅ 选材来源：你日常在用AI做的事 + 大多数人的共同痛点
- ✅ 每篇标题要让不写代码的人也愿意点
