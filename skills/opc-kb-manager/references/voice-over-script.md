# 口播稿写作规范

> 将公众号文章转换为口播稿（Voice-Over Script）的规范。

## 文章 vs 口播稿的区别

| 文章版 | 口播版 |
|--------|--------|
| 长段落 | 短句分行，一眼看到断句位置 |
| ▍/✦ 等视觉分隔 | 去掉，靠语气和内容自然过渡 |
| 书面语 | 口语化，"你懂的""坦白说""我就好奇了" |
| 引用框 > | 改成自然叙述 "xxx说过..." |
| 排版标记 | 全部去掉，纯语音稿 |

## 核心原则

1. **每行一个呼吸** — 每句一行或两行，让读的人自然断句
2. **去掉所有视觉元素** — 图片标记、分隔线、章节符号全部删除
3. **增加口语化过渡** — "所以"、"但是"、"你想想"、"不卖关子了"
4. **引号和强调** — 关键概念用「」括起来，录音时加重语气
5. **结尾留互动钩子** — "评论区聊聊"、"你怎么看"

## TTS 停顿控制（重要 — VoxCPM 和其他 TTS 通用）

**不要用自定义停顿标记。** TTS 不认识 `【停顿】` `【长停顿】` `<break>`，会当文字读出来。

### 正确做法：用标点符号控制停顿

| 标点 | 效果 | 使用场景 |
|------|:----:|---------|
| 。 | 长停顿 ~0.5s | 句子结束、观点切换 |
| ， | 短停顿 ~0.2s | 句中自然呼吸 |
| —— | 顿挫感 | 解释、转折、强调前 |
| ？ | 升调+停顿 | 设问、引发思考 |
| 空行 | 段落过渡 | 话题切换 |

### 口播稿准备流程

```
公众号文章 → 改写短句口播稿（每句一行）
                       ↓
检查每段结尾是否有 。—— 这就是停顿点
                       ↓
在 VoxCPM 官网试听，如果节奏太快
                       ↓
在需要更长的停顿处把逗号改成句号
```

### 声音描述写法

描述要具体到可感知的维度，不要写模糊指令：

```
✅ (male, warm and confident, slightly faster, like a podcaster talking to a friend)
❌ (带一点感情，注意停顿，语气自然一点)
```

## 示例转换

原文：
```
▍ 先搞清楚一个概念：一人公司到底是什么？

我开始搜索"一人公司"这个词。发现满屏都是"月入 5 万""裸辞翻身"。
但仔细想想，有个问题得先理清楚：一人公司跟个体工商户有什么区别？
```

口播版：
```
那有个问题得先理清楚：一人公司跟个体工商户到底有什么区别？

我开始搜这个词。
满屏都是月入五万、裸辞翻身、一个人干翻一个团队。

但我发现有个问题得先理清楚。
一人公司跟个体工商户，到底有什么区别？
```
