# 简笔画生成工作流 (Line Art Generation)

> 用于自媒体品牌形象的简笔画配图生成。
> 品牌形象：英短蓝猫（固定主角），每张图匹配文案场景。

## 需求总结

用户需要一个品牌形象（英短蓝猫），根据口播/文案内容生成对应的简笔画配图。

**风格要求：** 像用一支笔画出来的手绘感，黑白线稿，白底，不要填充色。

## 方案对比

| 方案 | 费用 | 风格 | 速度 |
|:----:|:----:|:----:|:----:|
| 扣子(Coze)智能体 | 免费额度有限，充值较贵 | 手绘感好 | 48s/张 |
| 硅基流动 Z-Image-Turbo | 免费 | 有颜色填充，需后处理 | 8s/张 |
| 硅基流动 + PIL转线稿 | 免费 | 可调出纯线稿 | 8s+1s |

## 推荐方案：硅基流动 + PIL 线稿处理

### Step 1: 生成干净底图

用硅基流动 `Tongyi-MAI/Z-Image-Turbo`（免费模型）生成纯白底图。关键：prompt 强调"纯白背景，无任何背景元素，画面干净，主体突出"。

### Step 2: 用 PIL 转黑白线稿

```python
from PIL import Image, ImageFilter, ImageOps, ImageEnhance

img = Image.open("input.png").convert("RGB")
gray = img.convert("L")
edges = gray.filter(ImageFilter.FIND_EDGES)
edges = ImageEnhance.Contrast(edges).enhance(3)
bw = edges.point(lambda x: 255 if x > 60 else 0)  # 阈值控制线条密度: 40密→120疏
lineart = ImageOps.invert(bw)
lineart.save("output.png")
```

### 对话框文字

在 prompt 中描述对话框和文字，AI 画出后一并转线稿。

### Prompt 模板（硅基流动）

```
[固定主角: 一只英短蓝猫，圆脸大眼睛灰色毛发] + [动作/场景] + [情绪] + [纯白背景，主体突出]
```

实测有效的各章节 prompt（一人公司文章为例）：

| 章节 | Prompt |
|------|--------|
| 失业迷茫 | 一只英短蓝猫垂头丧气坐在空荡荡的办公桌前，桌上只有一个空杯子，周围很冷清 |
| 概念辨析 | 一只英短蓝猫面前摆着三个路牌分别写着"个体工商户""一人公司""超级个体"，猫歪着头思考 |
| 自媒体套路 | 一只英短蓝猫拿着放大镜看手机屏幕，屏幕上有"月入5万""教你赚钱"字样，猫一脸怀疑 |
| Dan Koe框架 | 一只英短蓝猫趴在本子上画路线图，笔记本上画着箭头和步骤数字 |
| AI系统 | 一只英短蓝猫坐在驾驶舱里看多个屏幕 |
| 结尾希望 | 一只英短蓝猫迎着阳光大步向前走 |

注意：硅基流动 Z-Image-Turbo 无法直接生成线稿文字（中文字符会乱码或缺失）。对话框文字可在后处理时用图片编辑软件添加。

### 线稿阈值调优

PIL FIND_EDGES 后的二值化阈值控制线条密度：

| 阈值 | 效果 | 文件大小 |
|:----:|------|:--------:|
| 40 | 线条最多，细节保留多 | ~70KB |
| 60 | 适中，推荐默认 | ~68KB |
| 80 | 线条减少，更干净 | ~66KB |
| 120 | 线条最少，最简约 | ~61KB |

建议从 60 开始，根据图片复杂度和效果调整。

### 调用方式

A. **硅基流动API**（推荐，免费）→ 后处理转线稿
B. **扣子Coze智能体**（需充值，出图无需后处理）

## 输出规格

- PNG, 1024x1024, 处理后约20-70KB
- 路径: `D:\opc-workdown\outputs\sf-<文章名>-<章节>.png`

## 已知问题

- Z-Image-Turbo无法直接出纯线稿（总有颜色填充），需PIL后处理
- PIL边缘检测方案可用但不细腻。增强方案：ControlNet Canny + SDXL（需付费模型或本地GPU）
- 扣子充值后可以直接出纯线稿，效果更好
